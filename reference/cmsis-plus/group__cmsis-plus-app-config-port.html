Detailed Description

Definitions used to select various custom RTOS port implementations.

The CMSIS++ RTOS API can be configured to run on top of another RTOS scheduler (like FreeRTOS). It is also possible to select each synchronisation objects to use either the custom port objects or the CMSIS++ reference C++ implementations.

#define OS_USE_RTOS_PORT_SCHEDULER

Use a custom scheduler implementation.

When using a CMSIS++ port that runs on top of another RTOS, this option disables the CMSIS++ reference scheduler and forwards all related calls to the custom implementation RTOS.

The default is to use the CMSIS++ reference scheduler.
